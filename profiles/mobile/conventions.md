# Mobile Profile

## Assumptions

- Follow the project's existing state management approach.
- Keep UI, state, domain, and data layers separated when the project has that shape.
- Dispose controllers, subscriptions, and listeners.
- Validate common device sizes and loading/error states.
- Record meaningful mobile work in `docs/ai/feature-log/`.

## Agent Notes

- Confirm whether the project is Flutter, React Native, native iOS, or native Android before editing.
- Use emulator/device validation when practical.
