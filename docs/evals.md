# Evals

## Purpose

Evals prove whether the AI setup is getting better or worse.

Run them after meaningful kit changes.

## Scoring

Use 0-2 for each criterion:

- `0`: failed
- `1`: partial
- `2`: passed

## Common Criteria

- found the right files
- understood project conventions
- avoided unrelated changes
- used the correct workflow
- delegated to appropriate roles
- implemented the requested behavior
- ran relevant validation
- recorded feature/error/review cases when needed
- produced a concise final summary

## Result Location

Inside each project:

```text
docs/ai/evals/YYYY-MM-DD-kit-version.md
```
